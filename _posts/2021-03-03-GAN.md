# Generative Adversarial Nets  
> 본 논문을 개인의 해석 및 이해를 바탕으로 정리 



## 논문 원본  
[GAN](https://arxiv.org/pdf/1406.2661.pdf)  

## 논문 요약  

<h3 style="color: royalblue; font-weight: bold">논문 목적</h3>   

sample를 만들거나 training하는 동안에 **Markov chain**와 **inference**가 사용되지 않으면서 **Adversarial Process**를 통해서 generative model를 만드는 새로운 framework를 소개하기 위해서다.  

<h3 style="color: royalblue; font-weight: bold">Generative model, Discriminative model</h3>  

<img width = "400" src = "https://i.imgur.com/6ZPdsM8.png">  

#### Generative Model(Generator)  

데이터 X가 생성되는 과정에서 $$P(Y), P(X\mid Y)$$를 베이지안을 사용하여 $$P(Y\mid X)$$를 간접적으로 도출하는 모델을 나타낸다. 전자의 확률 모형에서 볼 수 있듯이 label이 별도로 존재하지 않아도 generative model를 이용할 수 있다. label이 존재하지 않아도 된다는 장점이 존재하지만, discriminative model보다 가정이 많기 때문에, 가정이 맞지 않으면 성능이 떨어진다는 단점이 존재한다.  
가정이 잘 맞는다면 적은 데이터 셋에서도 좋은 성능을 보일 수 있다. 데이터 셋이 많아지면 discriminative model의 성능과 비슷해진다.  
  
논문에서는 generative model로 data sample를 만들려고 한다. 최대한 train data 확률 분포와 비슷한 분포를 형성하며 generate시켜려고 한다. 실제 train data가 실제 현금이라면 generative model로 생성한 sample를 위조 지폐라고 생각하면 이해하기 쉽다.   

#### Discriminative model(Discriminator)    

데이터 X가 주어졌을 때 Label Y를 얼마나 잘 맞추는지 구별하는 모델이다. $$P(Y\mid X)$$를 직접적으로 반환하기 때문에, label이 필요하다. supervised learning 범주에 속한다. generative model보다 가정이 단순하고, 데이터 양이 많으면 충분히 좋은 성능을 낼 것이다.  

논문에서는 generative model이 생성한 fake data와 실제 데이터 셋(train data)에서 나온 data를 잘 구별하도록 discriminative model를 구축한다. Discriminative model은 경찰의 역할로 위조 지폐(generative model)과 실제 현금(real data)를 구별하는 역할이다.  

<h3 style="color: royalblue; font-weight: bold">Adversarial Net</h3>  

위의 z(**random noise**)를 multilayer perceptron으로 foward propagation시키면 data sample이 추출된다. 단지, 딥러닝에서의 mlp만을 이용하여 sample를 추출 수 있다. 그리고 Discriminative model도 Multilayer perceptron으로 구성되어 있어, forward propagation시키면 $$D(x;\theta_{d})$$, real data로부터 나왔는지의 확률을 구할 수 있다.  

<h3 style="color: royalblue; font-weight: bold">Value function 설명</h3>  

$$ \underset{G}{min} \underset{D}{max}V(D,G) = \mathbb{E}_{x~p_{data}(x)}[logD(x)]+ \mathbb{E}_{z~p_{z}(z)}[log(1-D(G(z)))]$$  

위의 식을 해석하면 D(discriminative)는 최대한 training sample과 generative sample를 구별할 확률을 최대화시키려고 하고, G(generative)는 $$log(1-D(G(z)))$$를 최소화시켜려고 한다. $$D(G(z))$$를 최대화 시키게되는데, 이는 fake sample를 최대한 real sample처럼 train시킨다는 의미이다.  

실제로 train 초반에는 $$D(G(z))$$를 Maximize시키며 Generator를 train시킨다. 초반에는 generative model이 추출한 sample은 터무니 없이 이상할 것이다. 이를 초반에 빠르게 update시키기 위해서는 gradient를 크게 해줘야 한다. $$log(1-D(G(z)))$$보다는 $$log(D(G(z)))$$가 gradient를 더 크게 하기 때문에 후자의 식을 이용한다.  
* log(1-x)에서 x가 0에 가깝게 나오면 기울기가 작고, log(x)에서 x가 0에 가깝게 나오면 기울기가 매우 커진다.  


**Train**할 때는 Discriminative k번, Generative 1번 번갈아가며 한다. 만약 Discriminative model를 최대한 optimizing시키면 generative model에서의 데이터는 한정된 data에 overfitting이 될 뿐만이 아니라, computational cost가 높다. k번, 1번씩 번갈아가며 완전히 Discriminative를 optimizing하는 것이 아니라 가깝게 optimizing시킨다.  

<h4 style="color: royalblue; font-weight: bold">Value function 설명</h4>   

<img width = "600" src = "https://user-images.githubusercontent.com/55014424/109826722-6a88b600-7c7e-11eb-9a38-161f45ecd6fe.png">  

a) 초반에 $$p_{g}$$와 $$p_{data}$$는 비슷해지면서 D는 일부분만 정확하게 분류할 것이다.  
$$p_{g}$$: generative model이 추출한 sample의 분포  
$$p_{data}$$: real data sample에서의 분포  

b) Discriminative model이 실제 data와 fake data를 구별시키며 train될 것이다. discriminative는  

$$D^{*}(x) = \frac{p_{data}(x)}{p_{data}(x)+p_{g}(x)}$$  

로 수렴시키며 train된다.   

c) Discriminative model를 train한 후, Generative Model를 train시키면, $$p_{g}$$(녹색 선)은 $$p_{data}$$(검은색 점)으로 가까워진다.  

d) 몇 번 train과정 뒤에는, generative model의 fake sample 분포와 real data sample 분포가 거의 동일해질 것이다. 파란색 선인 $$D(x)$$은 1/2로 수렴하며 fake sample과 real data sample를 구별 못하게 된다.

<h3 style="color: royalblue; font-weight: bold">GAN 대표 문제</h3>  

<img width = "400" src = "https://i.imgur.com/l9sDQK6.png">  

파란색 선이 real data의 분포라고 하자. real data가 여러 분포를 갖고 있으면 GAN은 여러 분포 중 하나의 분포를 선택해서 비슷해지도록 train될 것이다.  
오른쪽 사진처럼 분포가 2개가 존재할 때, GAN은 하나의 분포와 닮아지도록 train된다.  
즉, train data의 여러 분포의 다양성을 cover하지 못한다는 문제가 있다.
